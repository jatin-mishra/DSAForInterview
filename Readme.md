
# How to approach in Interview
- Approach like design problem
- Understand Requirements
- Discuss Corner cases
  - What to do if input is x and more.
- Start Solutioning First Approach, explicit Base cases.
- Discuss Time and Space Complexity
- Optimize if possible
- You don't need to remember lema's, tricky questions which can be solved if known one specific lema 
- Interviewer wants to test to thinking process and problem solving skills, not to test your memory.


# Example Structure 
- Requirement 
- Corner cases 
- First Approach, Base cases discussion
- Explain time and space complexity 
- Optimizations and Iterations
